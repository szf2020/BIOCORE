## 一、产品概述

### 1.1 产品定位

BIOCore 是面向非GMP实验室R&D环境的**生物反应器智能控制平台**，基于 S7-200 SMART G2 PLC + Node.js 全栈架构，目标用户为高校发酵实验室、生物科技初创公司和合成生物学研究团队。

| 维度 | Eppendorf TruBio | Sartorius MFCS | **BIOCore** |
|------|-----------------|----------------|-------------|

| 数据库 | 专有格式 | SQL Server (需许可) | **InfluxDB + SQLite，零许可费** |

| 并行反应器 | 最多8台 | 最多32台 | **8台/PC，多PC可扩展** |

## 二、核心功能模块

### 2.1 PLC通讯驱动

**多协议支持**
- S7协议 (node-snap7)：工业级 Snap7 C库，支持 S7-200/300/400/1200/1500 全系列
- Modbus RTU/TCP (modbus-serial)：变频器RS232通讯，读取转速/电流/故障码
- 灵活地址格式：`DB2.DBW4`、`DBW4`、`VW100`、`V200.0` 多种格式自动识别

**双向心跳协议**
- PC→PLC：每秒递增计数器 (0~65535)，PLC监测3秒不变→安全驻留
- PLC→PC：PLC每秒递增，PC监测3秒不变→自动Hold + 操作员手动确认恢复
- 断线期间：补料全停、搅拌降速200rpm、温度/pH PID继续维持

**安全特性**
- 地址校验：前后端双重验证，禁止无效地址写入PLC
- 只读测试：变量测试操作绝不向PLC写入任何数据
- BOOL写入验证：read-modify-write + read-back确认
- 异步互斥锁：防止并发PLC读写冲突

### 2.2 8罐并行控制

**多反应器管理**
- 单台PC同时控制最多8个独立发酵罐 (F01~F08)
- 每个罐有独立的PLC连接、批次状态、配方和数据流
- Dashboard顶部罐选择栏，一键切换罐视图
- 每个罐独立选择配方、下载、启动、控制

**反应器状态面板**
```
[F01 ● RUN] [F02 ● IDLE] [F03 ● PAUSE] [F04 ● IDLE]
[F05 ● IDLE] [F06 ● IDLE] [F07 ● IDLE] [F08 ● IDLE]
```

### 2.3 ISA-88 状态机

**六状态标准流转**
```
Idle → Running → Complete
         ↕           
       Paused    
         ↕
       Held (故障自动触发)
         ↓
      Stopped ← E-STOP (任意状态)
         ↓
       Idle (复位)
```

**按钮使能矩阵**

| 按钮 | Idle | Running | Paused | Held | Stopped |
|------|------|---------|--------|------|---------|
| 启动 | ✅(有配方) | — | — | — | — |
| 暂停 | — | ✅ | — | — | — |
| 恢复 | — | — | ✅ | ✅ | — |
| 放弃 | — | — | ✅ | ✅ | — |
| 急停 | ✅ | ✅ | ✅ | ✅ | — |
| 复位 | — | — | — | — | ✅ |

**per-Phase独立状态机**
- 每个Phase由操作员手动决定启动/跳过
- Phase完成后不自动推进，等待操作员确认
- Phase状态：待执行 → 就绪 → 运行中 → 已完成/已跳过/已保持

### 2.4 配方管理系统

**配方全生命周期**
- 创建 → 编辑 → 锁定(审批) → 下载到罐 → 执行 → 归档
- 只有已锁定(approved)的配方才能下载到罐中执行
- 版本管理：每次修改生成新版本，旧版本自动标记为superseded

**拖拽式配方编辑器**
- 14种Phase模板库（系统操作/温控/过程控制/发酵/清洗灭菌）
- @dnd-kit 拖拽排序
- 每种Phase可配置步骤序列、完成条件、PLC变量关联
- JSON导入/导出
- 5种预置配方示例（E.coli/酵母/CHO/CIP/DoE对照）

**Phase模板配置**
- 14种内置Phase完全可编辑
- 每个Phase可自定义：步骤名称、完成条件(AND/OR/NOT组合)、跳转目标
- 参数直接绑定PLC变量（从已配置的PLC变量列表选择）
- 支持新建自定义Phase类型

### 2.5 清洗灭菌独立控制

**CIP (就地清洗)**
- 独立页面和状态机
- 5步清洗循环：碱洗→水洗→酸洗→水洗→终点检测
- 电导率判定终点

**SIP (就地灭菌)**
- 4步灭菌序列：预排空气→升温121°C→保温F₀积分→冷却
- F₀值实时积分，冷点温度低于100°C自动重置
- 安全互锁：主发酵运行中禁止启动CIP/SIP

### 2.6 数据记录与管理

**双数据库架构**

| 数据库 | 存储内容 | 技术 |
|--------|---------|------|
| InfluxDB | 过程数据(温度/pH/DO/转速/流量)、软测量值、SV值 | 列式压缩，Flux查询 |
| SQLite | 批次记录、配方、审计日志、报警、校准、Phase/Step执行日志 | WAL模式，零运维 |

**数据采集器**
- 每秒采集PLC过程值→WebSocket实时推送→Dashboard即时显示
- 每分钟聚合→InfluxDB持久化存储
- 自动计算：OUR(耗氧速率)、kLa(传质系数)、μ(比生长速率)、F₀(灭菌值)

**不可篡改审计日志**
- 所有参数修改和状态跃迁不可篡改记录
- SQLite触发器禁止UPDATE和DELETE操作
- 操作员ID、时间戳、变更前后值完整记录

### 2.7 运行时故障检测

**11项RunningFault自动监测（每秒检查）**

| 编号 | 故障 | 条件 | 动作 |
|------|------|------|------|
| RF-01 | 变频器故障 | 故障码≠0 | 搅拌急停→Held |
| RF-02 | VFD通讯中断 | 读取失败≥3秒 | 搅拌急停→Held |
| RF-03 | 温度偏差 | >2°C持续>3min | Held |
| RF-04 | pH偏差 | >0.5持续>5min且泵满速 | 补料泵停→Held |
| RF-05 | DO过低 | <5%持续>5min | 补料泵停→Held |
| RF-06 | 罐压过高 | >2.5bar | 排气阀全开→Held |
| RF-07 | 传感器断线 | 原始值<100 | 记录故障→Held |
| RF-08 | 传感器饱和 | 原始值>28000 | 记录故障→Held |
| RF-09 | 称重无变化 | 30min无变化但泵运行 | 补料泵停→Held |
| RF-10 | 泡沫事件 | 称重突增>5%无泵动作 | 消泡泵脉冲 |
| RF-11 | PC-PLC通讯断线 | 心跳停滞≥3秒 | 自动Hold+补料停 |

## 三、AI智能功能

### 3.1 本地LLM集成 (Ollama)

**本地优先，数据不出境**
- 集成 Ollama 本地大语言模型（推荐 Qwen2.5-7B）
- 一键部署，CPU模式即可运行（推荐32GB内存）
- 可选云端API回退（Anthropic/OpenAI）

**AI助手面板**
- 聊天式交互界面
- 快捷问题按钮：
  - "当前DO趋势分析"
  - "本批次与历史对比"
  - "最近异常原因"
  - "补料速率建议"

### 3.2 自然语言查询 (NL→Flux)

**用中文问问题，自动生成数据库查询**

操作员输入：
> "最近24小时F01罐的温度和pH变化趋势是什么？"

系统自动转换为 InfluxDB Flux 查询：
```flux
from(bucket: "fermentation")
  |> range(start: -24h)
  |> filter(fn: (r) => r.batch_id == "BATCH-20260406-001")
  |> filter(fn: (r) => r._field == "temperature" or r._field == "pH")
  |> pivot(rowKey: ["_time"], columnKey: ["_field"], valueColumn: "_value")
```

### 3.3 批次摘要自动生成

批次完成后，本地LLM自动生成200-500字的结构化中文摘要：

> **批次摘要 — BATCH-20260406-001**
> 
> **培养概况：** E.coli BL21高密度补料培养，37°C，pH 7.0，DO 30%级联控制。总发酵时间18.5小时，分8个Phase执行。
> 
> **关键工艺事件：** 8小时DO Spike检测到碳源耗尽，切换至指数补料阶段。补料期μ维持在0.15/h...
> 
> **异常与处理：** 12小时出现RF-05 DO过低报警，自动Hold后操作员确认恢复...
> 
> **结论建议：** 最终OD600=25.3，建议下批次提高初始补料速率...

### 3.4 CUSUM实时异常检测

**基于历史批次包络线的在线偏移检测**
- 从≥5个历史批次建立基线（均值±2σ）
- 每分钟运行CUSUM算法检测系统性偏移
- 超过阈值触发软报警（橙色弹窗提示）
- 支持温度/pH/DO/搅拌/罐压多参数独立监测

### 3.5 DTW批次相似度匹配

**动态时间弯曲(DTW)算法**
- 选定目标批次后，自动计算与所有历史批次的DTW距离
- 相似度排名：找到最接近的"黄金批次"
- 多批次叠加曲线对比显示
- 按发酵经过时间自动对齐

### 3.6 软测量与补料建议

**线性回归在线推断（可升级ONNX）**
- 基于温度/pH/DO/转速/通气等在线数据推断OD和残糖浓度
- Dashboard显示AI估算值（虚线+95%置信区间）
- 超出训练范围时自动标记"外推警告"

**Monod动力学补料优化**
- 基于软测量推断的实时生物量
- 结合Monod动力学参数(μmax/Ks/Yxs)计算最优补料速率
- 弹窗推荐补料调整，需操作员手动确认（AI永远不直接写入PLC）

### 3.7 异常根因分析

**报警触发后自动分析**
- 提取报警前后30分钟的多参数趋势
- 基于已知故障模式库匹配可能原因
- 按置信度排序列出：
  - 可能原因 + 证据 + 置信度
  - 受影响参数
  - 处理建议
  - 事件时间线叙述

### 3.8 贝叶斯实验设计优化

**高斯过程 + UCB采集函数**
- 基于所有历史批次终点指标（如OD、产物效价）
- 推荐下一批次的最优参数组合（温度/pH/补料速率等）
- 自动平衡探索(exploration)与利用(exploitation)
- 支持多保真度优化：摇瓶→发酵罐数据融合

### 3.9 增量学习

**每完成一个新批次自动更新模型**
- OLS回归模型在线重训练
- R²值实时追踪模型精度
- 数据积累越多模型越准确
- 完全本地运行，零云端依赖

## 五、安全体系

### 5.1 PLC安全

- **双向心跳**：PC↔PLC双向监测，任一方断线3秒内检测
- **安全驻留**：通讯断线期间PLC独立维持安全状态
- **硬件互锁**：蒸汽/冷却阀不同时开启（PLC硬逻辑）
- **急停**：任何活动态均可触发，直接→Stopped，不可恢复只能复位

### 5.2 软件安全

- **SQL注入防护**：白名单列名校验 + 参数化查询
- **Flux注入防护**：输入验证 + 字段白名单 + 正则校验
- **地址校验**：前后端双重验证，14种合法地址模式
- **AI安全缓冲**：AI建议永远不直接写入PLC，必须经操作员确认
- **审计不可篡改**：SQLite触发器禁止UPDATE/DELETE

### 5.3 访问控制

| 角色 | 权限 |
|------|------|
| admin | 全部权限 + 用户管理 |
| engineer | 配方编辑 + PID参数修改 + 校准 |
| operator | 批次启停 + 参数查看 + AI建议采纳 |

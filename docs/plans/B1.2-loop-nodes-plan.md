## 2. Risk Areas

### 2a. Edge cases

1. **`exitExpression` parse error or PV missing.** Spec 静默；commit-1 计划：throw → take body 继续.IF/ELSE 先例（`dag-executor.ts` line 200-203）使用 `default_branch ?? 'false'`. **Open Question (i).**
2. **User wires back-edge to wrong target.** BV-25 验自有的 back-edge 至少一条；BV-15 cycle detection 抓其他 stray cycle. 多条 back-edge 至 loop 节点是无害的（每条 force 一次迭代检查）. 无额外规则.
3. **Body 含 0 phase 节点（degenerate）.** 一个仅 branch+goto 的 busy-loop body 会被 visitCount auto-ceiling cap. 可加 BV-25b（warning）但 out of scope.
4. **Loop node 是 start 直接后继.** First entry 在 `dagExecutor.start() → advanceToPhaseOrEnd()` 中前 hit loop 节点；commit 1 已加 `'loop'` 到 skip-through 集合.
5. **Nested loops 在 exit subgraph（post-loop）.** BV-24 仅检 body 子图；exit 后再来一个 loop 是顺序非嵌套，允许.
6. **Body 直接 terminate at end node（无 back-edge）.** BV-25 抓.
7. **`maxIterations: 0`.** BV-22 拒（`>0` 必须）.
8. **`maxIterations: 1`.** First entry push iter=0；body 跑；back-edge 进入；iter→1；`1>=1` exit. 正确.

### 2b. 与现存代码交互

1. **BatchController state machine（XState）.** 不受影响（仅 batch-level state）. 无集成风险.
2. **F2-AUTO `resumeAndStart()`.** 最高风险.`resumeBatch()` → `dagExecutor.start()` 清空 frames（line 175-178）. 计划：加 `savedFrames` 参数并在 `start()` 后 `restoreFrames(savedFrames)`.5 个 call site（startup.ts + tests）必须更新.TS 编译会全 catch.
3. **audit-queue.** 新 event 类型 `loop_*` 加 ~3 audit/loop iteration. 100 iter loop = 300 rows，远低于 ~50 audit/sec capacity. queue 是 schema-agnostic 在 `action`，无变更.
4. **WS 带宽.** 1Hz polling × 100 iter × 100 min = 200 messages overhead，可忽略.
5. **realtime store on web-ui side**（`packages/web-ui/src/types/index.ts` line 250 `BatchRuntimeState`）：可加 `LoopIterationEntry` for live timeline. **Out of scope for B1.2**（UI consumption 是 follow-up）；audit table 已捕全.
6. **现存 `branch_evaluated` event vs new `loop_iterated`.** 独立事件 — loop body 内 branch 仍 fires `branch_evaluated`. operator 在 timeline 看到两者，正确.

### 2c. Migration 024 backward-compat

- 旧 DB 无列：迁移加列为 NULL → `getBatchLoopFrames` 返 null → 控制器视作"无活跃 loop"，与 fresh-start 不变量一致.
- 新 DB：从 baseline (001) 逐步 → 024，列在任何代码读取前已建.
- 任何代码经 helper 读取，helper 处理 NULL/parse 错误优雅 — 旧 DB 启动不崩.

### 2d. v1.10.0 LoopFrame 字段 — required vs optional for B1.2

| Field | v1.10.0 | B1.2 | 备注 |
|-------|---------|------|------|
| `loopNodeId` | required | required | unchanged |
| `iteration` | required | required | unchanged |
| `maxIterations` | optional | optional | spec Q2 至少一个 required at validator |
| `exitExpression` | optional | optional | 同上 |

## 4. Rollback Strategy

### Per-commit `git revert` impact

- **Commit 4 reverted:** Editor 不能创建新 loop recipes；现存 loop recipes 渲染为默认 react-flow box（仍可执行）.Audit/WS 事件消失.客户端 validator 退化到服务端.**Operationally inert.**
- **Commit 3 reverted:** Migration 024 列残留 DB（rolling back 文件不 drop 列）.`_migrations` 行删除会导致下次 migrate 试 re-add 失败 `duplicate column`.**Mitigation:** safe revert script 必须 `DELETE FROM _migrations WHERE name = '024-batch-loop-frames'` 但**不 drop 列**.列在无代码读时 harmless. 控制器 persist hook 没了 → 中途 crash 失 frame state，resume 从 saved currentNodeId 起步无 iter context（loop 重 enter at iter 0 — 一个 known consistency gap 但终止 via maxIterations/exitExpression）.**Acceptable degradation.**
- **Commit 2 reverted:** Validator 不再拒 malformed loops；executor 仍跑 commit 1 逻辑.无 exit 无 max 的 loop 在 auto ceiling max(0, 1000)=1000 处 cap.**Risky** — 野外 malformed recipe 走更远再被抓.
- **Commit 1 reverted:** 全 loop-aware 代码消失.含 `'loop'` 节点的 recipe 会在 executor `advance()` 抛错（type-narrowing else 分支不 match）.**Highest-impact revert.** 必须 4→3→2→1 级联.

### Migration 024 down-migration

按项目惯例（`migrator.ts` line 92 `down: throw new Error("不支持回滚")`）不提供 down migration. 列 TEXT NULL 默认 NULL，仅经 try/catch helper 读. **Operational impact post-revert：零.** 存储成本：NULL → 0 字节 per row in SQLite.

### Recommended rollback recipe

如 B1.2 必须 post-deploy 回退：
1. Pause server.
2. `git revert <commit-4>..<commit-1>`（4 commits, reverse order）
3. **不要** drop `current_loop_frames` 列.留 NULL.
4. **不要** delete `024-batch-loop-frames` from `_migrations`. 留 as run.
5. 重启服务.现存 batches resume normal；新 batch 只 linear+branch+goto.

## 5. Anti-Spec Items（out of scope）

明确 deferred — 出现在 v1.10.0 LoopFrame design space 但 B1.2 不实现：

1. **嵌套 loops（loop in loop）.** Spec Q5 fixes depth=1.**Unblocked by:** drop BV-24，per-frame iteration in audit（已 JSON-serialized）+ careful resume — `restoreFrames` 已是 stack，无 work needed there. Validator-level only. 留待真实生物工艺需求驱动（很可能永不需要）.
2. **Time-bounded loops**（`startedAt` + `maxDurationMs`）.LoopFrame 字段已存但 B1.2 executor 永不读. **Unblocked by:** 加第三个 end-of-iteration check（`Date.now() - startedAt >= maxDurationMs → exit`）+ 第三个 optional Loop 节点字段. ~30 LOC. 留 B1.5.
3. **Convergence-detection loops**（如 "exit when OD600 has been within ±0.1 of last sample for 3 consecutive samples"）. 需 history beyond current PV snapshot. **Unblocked by:** persist per-frame state beyond iteration counter（如 `state: Record<string, number>` field on LoopFrame）.Major.
4. **Loops as Subrecipe building blocks**（B1.4 territory）.Subrecipe 是单独节点类型 push stack frame to invoke 子 DAG. Loop+Subrecipe 都用 LoopFrame stack 但 disjoint frame shapes（loop 有 `iteration`，subrecipe 有 `subRecipeId`+`parentNodeId`）.栈 generic; B1.4 加 own discriminated-union type.
5. **Loop UI simulation/preview.** 编辑器无 "simulate loop with mock PVs". 操作员必须信 BV-22..25. **Unblocked by:** 加 dry-run executor mode 用 faked `evaluateExpression`. Out of scope.
6. **Loop iteration timeline visualization.** Web-ui batch detail 不渲染 loop iter 历史. Audit log 已捕；无图. **Unblocked by:** `<LoopIterationTimeline>` reading new `loop_*` events. Follow-up.
7. **Adaptive `maxRevisits`.** 计划 auto ceiling = `max(maxIterations, 1000)`. `maxIterations=10000` 的 loop 设 ceiling=10000 但仅当 caller 未传更大显式值. 多 loops 顺序共存 → ceiling = max of all.无需 B1.2 work，但 commit 1 test #8 覆盖.
8. **服务端 `loop_*` audit retention policy.** audit_logs 当前无 TTL.1000-iter loop 产 2002 rows. Defer; 任何用户实际 hit 才上 ops issue.

## 一、产品定位与一句话描述

**BIOCore — 实验室 R&D 发酵罐智能控制平台**

一句话：用工业 PLC 的可靠性 + 现代 Web 技术的易用性 + 本地 AI 的智能化，以竞品 **1/5 的成本**为实验室提供专业级发酵过程控制。

**目标用户：**
- PI（课题组负责人）：关注成本、数据完整性、合规审计
- 工艺工程师：关注控制精度、多罐并行、配方灵活性
- 合成生物学研究人员：关注 AI 辅助优化、实验设计、数据分析

**适用场景：**
- 非 GMP 实验室 R&D 发酵（微生物、细胞培养）
- 1-50L 台式发酵罐（玻璃罐体、不锈钢罐体均适用）
- 大肠杆菌、酵母、CHO 细胞等常见表达系统

## 三、核心功能模块（建议作为 PPT 的主体章节）

### 3.1 PLC 通讯层 — 工业级可靠性

**硬件基础：西门子 S7-200 SMART G2 PLC**
- 8 路模拟输入（AI）：温度 PT100、pH、DO、罐压、气流、称重
- 4 路模拟输出（AO）：蒸汽阀、冷却阀、空气阀、预留
- 14 路数字输入（DI）：急停、阀位确认、盖锁、压力开关
- 10 路数字输出（DQ）：4 台蠕动泵（补料/补碱/补酸/消泡）+ 电磁阀
- 8 路 PID 回路：温度、搅拌、DO、pH、罐压等独立闭环控制
- VFD 变频器：RS485 Modbus RTU 通讯，搅拌转速无级调速

**双向心跳安全机制：**
- PC 每秒向 PLC 写入递增计数器（VB400）
- PLC 每秒向 PC 回写确认计数器（VB401）
- 任一方 3 秒无响应 → 自动触发安全保护
- PLC 侧独立安全连锁：即使 PC 崩溃，PLC 仍保持搅拌 + 温控 + pH 控制

**通讯协议支持：**
- S7 协议（node-snap7）：主通讯链路
- Modbus RTU/TCP（modbus-serial）：变频器 + 扩展设备
- 自动重连：指数退避策略（1s → 5s → 30s）

### 3.2 ISA-88 批次控制引擎

**工业标准状态机（6 状态）：**
```
闲置(Idle) → 运行(Running) → 完成(Complete)
                ↕                    ↓
            暂停(Paused)          重置(Reset)
                ↕                    ↓
            保持(Held) → 停止(Stopped) → 闲置
```

**14 种 Phase 类型覆盖全流程：**
| 类别 | Phase 类型 | 说明 |
|------|-----------|------|
| 系统操作 | 准备(Prepare)、加水(Water Fill)、手动加料(Manual Add)、排液(Discharge) | 批次前后操作 |
| 温控 | 升温(Heating)、温度控制(Temp Control) | 夹套加热/冷却 |
| 过程控制 | 搅拌(Agitation)、补料(Feeding)、pH 控制、DO 控制、通气(Aeration) | 核心工艺参数 |
| 发酵主体 | 发酵(Fermentation) | 主发酵阶段 |
| 清洗灭菌 | CIP 在线清洗、SIP 在线灭菌 | F₀ 积分验证 |

**配方执行模式：**
- **顺序模式（Sequential）：** Phase 自动依次推进，适合成熟工艺
- **自由模式（Free）：** 操作员手动控制每个 Phase 启停，适合摸索阶段
- **DAG 分支模式（v2）：** 支持条件分支（如 "OD600 > 5 则切换补料策略"），实现智能工艺路由

**安全保护体系：**
- **启动前互锁（IL-01 ~ IL-10）：** 传感器信号、VFD 通讯、急停复位、盖锁、蒸汽压力等 10 项检查全部通过才能启动
- **运行故障监测（RF-01 ~ RF-11）：** 温度偏差 >2°C 持续 3 分钟、pH 失控 5 分钟、DO <5% 持续 5 分钟、罐压超限等 → 自动 Hold 保护
- **泡沫检测：** 称重突变 >5%（非泵料期间）→ 自动启动消泡泵
- **通讯断线保护：** PLC 心跳丢失 → 自动 Hold + PLC 独立运行安全程序

### 3.3 实时数据采集与存储

**双数据库架构：**
| 数据库 | 用途 | 特点 |
|--------|------|------|
| **InfluxDB** | 时序数据（温度、pH、DO 等过程参数） | 1 分钟聚合写入，365 天原始 + 永久降采样归档 |
| **SQLite** | 业务数据（配方、批次、审计、报警、取样） | WAL 模式高并发，不可篡改审计日志（数据库触发器保护） |

**实时数据流：**
```
PLC → 1Hz 采集 → WebSocket 实时推送（前端 <1s 延迟）
              → 60s 聚合 → 计算参数 → InfluxDB 持久化
```

**计算参数（实时推导）：**
| 参数 | 符号 | 说明 |
|------|------|------|
| 耗氧速率 | OUR | mmol/L/h，反映代谢活性 |
| 传质系数 | kLa | 1/h，Van't Riet 关联式 |
| 比生长速率 | μ | 1/h，OUR 对数微分 |
| 液体体积 | V_liquid | L，初始 + 累积补料 |
| 灭菌当量 | F₀ | min，SIP 效果验证 |
| 累积补料 | cum_feed | mL，蠕动泵积分 |
| 累积补碱 | cum_base | mL |
| 呼吸商 | RQ | CER/OUR，代谢状态指标 |

### 3.5 本地 AI 智能模块

**核心理念：100% 离线运行，零云依赖，AI 永不直接写 PLC**

**AI 技术栈：**
| 模块 | 技术 | 功能 |
|------|------|------|
| LLM 对话 | Ollama + Qwen2.5-7B（本地） | 自然语言查询数据、批次摘要生成、工艺问答 |
| 异常检测 | CUSUM 累积和控制图 | 实时监测参数偏移，提前预警（比传统阈值报警早 5-15 分钟） |
| 批次匹配 | DTW 动态时间规整 | 找到历史相似批次，辅助诊断和参数参考 |
| 软测量 | OLS 线性回归（v1）/ ONNX（v2 规划） | 在线推断 OD600、残糖浓度、产物浓度 |
| 补料优化 | Monod 动力学 + 指数补料 | 根据软测量估计的生物量，实时计算最优补料速率 |
| 根因分析 | 知识库模式匹配 | 报警触发时，自动分析 30 分钟前后参数趋势，定位可能原因 |
| 事件检测 | 统计特征识别 | DO Spike（底物耗尽）、滞后期结束、溢流代谢、稳定期进入 |
| 代谢推断 | 碱消耗速率分析 | 推断当前生长阶段（正常/溢流/底物耗竭/滞后/稳定） |

**AI 安全机制 — 建议缓冲区模式：**
```
AI 推理 → 生成建议（参数 + 置信度 + 理由）
       → 写入建议缓冲区（pending 状态）
       → 操作员审核（接受/拒绝）
       → 接受后才写入 PLC 设定值
```
AI 建议永远不会自动执行，必须经过人工确认。这是实验室安全的核心设计原则。

### 3.6 实验设计与优化（DoE）

**支持 7 种实验设计方法：**
| 方法 | 适用场景 | 特点 |
|------|---------|------|
| 全因子设计 | 因素 ≤4，全面交互分析 | 2^k 或 3^k 组合 |
| 中心复合设计（CCD） | 响应曲面建模 | 轴向点 + 中心点 |
| Latin 超立方采样 | 因素 >4，空间填充 | 均匀分布 |
| Plackett-Burman | 快速筛选关键因素 | N 次实验筛 N-1 因素 |
| Box-Behnken | 3 水平响应曲面 | 比 CCD 更少实验次数 |
| 确定性筛选设计（DSD） | 主效应 + 二次效应 | 2k+1 次实验 |
| 分数因子设计 | 大因素数简化 | 混淆主效应可控 |

**分析工具：**
- 极差分析（正交实验排名）
- 方差分析（ANOVA 显著性检验）
- 响应曲面拟合（线性/交互/二次）
- 田口信噪比（望大/望小/望目）
- 贝叶斯优化（高斯过程 + UCB 采集函数）
- 多保真度优化（摇瓶 → 发酵罐渐进验证）

### 3.7 MES 系统集成

**开放 API 体系：**
- REST API：97+ 端点，覆盖 PLC/配方/批次/报警/KPI/DoE 全部功能
- WebSocket：10 个实时频道（过程数据/状态变更/报警/AI 建议等）
- 双鉴权：JWT（前端，24h）+ API Key（MES，长期有效）
- 统一响应格式：`{code, msg, data, trace_id}` 支持跨系统追踪
- Swagger/OpenAPI 文档自动生成

## 九、典型应用案例（建议用于 PPT 的场景页）

### 案例 1：大肠杆菌 Fed-batch 发酵
- 配方：准备 → 加水 → 手动加料（培养基）→ 灭菌（SIP, F₀≥20）→ 升温至 37°C → 发酵（batch 阶段 6h）→ 触发补料（OD600>5）→ 指数补料 12h → 降温诱导（25°C, IPTG）→ 诱导 8h → 排液
- AI 辅助：CUSUM 检测 DO Spike → 软测量推断 OD600 → 自动建议切换补料策略

### 案例 2：酿酒酵母抗 Crabtree 效应
- DO-stat 补料控制：AI 监测 RQ 值，当 RQ > 1.1 时建议降低补料速率
- 事件检测：识别溢流代谢（碱消耗振荡），自动标记

### 案例 3：CHO 悬浮细胞培养
- 低搅拌（80-120 rpm）+ 高通气 + 严格 pH/温度控制
- 软测量推断活细胞密度，辅助工艺决策

### 案例 4：DoE 优化工艺参数
- Phase I：黄金分割法单因素搜索（22 次实验）
- Phase II：L9(3⁴) 正交试验 + 极差/方差分析
- Phase III：贝叶斯优化收敛最优条件（5 次确认实验）
- 总计约 47 次实验，14 周完成系统工艺优化

## 十一、安全设计亮点（建议作为专门的一页）

1. **AI 建议缓冲区：** AI 永远不能直接控制 PLC，所有建议必须操作员确认后执行
2. **双向心跳监测：** PC 和 PLC 互相监控，任一方故障立即保护
3. **PLC 独立安全连锁：** 即使上位机完全失联，PLC 仍维持搅拌（防沉降）、温控（防过热/过冷）、pH 控制
4. **不可篡改审计日志：** SQLite 触发器禁止 UPDATE/DELETE，所有操作永久记录
5. **启动前 10 项互锁检查：** 传感器、VFD、急停、盖锁、蒸汽供应等必须全部通过
6. **11 项运行故障自动保护：** 温度偏差、pH 失控、DO 过低、罐压超限、泡沫检测等
7. **F₀ 灭菌积分验证：** SIP 阶段实时计算 F₀ 值，确保灭菌效果达标
8. **安全表达式求值器：** 配方条件分支使用白名单 AST 解析，杜绝代码注入
9. **API 安全：** 时序安全比较（防时序攻击）、JWT 24h 过期、API Key 哈希存储
10. **文件上传校验：** MSDS 上传验证 MIME + 魔术字节 + 20MB 限制

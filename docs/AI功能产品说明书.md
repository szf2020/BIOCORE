## 1. 功能总览

BIOCore AI 模块提供从实时监控到智能决策的全链路 AI 能力, 覆盖发酵过程的异常检测、批次对比、自动建议、软测量推断和根因分析。

**核心原则: AI 永远不直接操控 PLC。** 所有 AI 输出均通过"建议缓冲区"呈现给操作员, 由人工审核后决定是否执行。

| 模块 | 功能 | 入口 | 数据源 |
|------|------|------|--------|
| AI 对话 | 自然语言查询发酵数据 | 侧栏 → AI助手 | Ollama / 云端 LLM |
| CUSUM 异常检测 | 实时监控5通道工艺偏移 | Dashboard 面板 | PLC 实时数据 (60s) |
| 批次智能分析 | DTW 相似度匹配 + 包络线对比 | 批次详情 → 智能分析 | InfluxDB 时序数据 |
| 自动建议 | 后台分析, 自动生成调整建议 | Dashboard 建议卡片 | 阈值 + CUSUM + FeedAdvisor |
| 软测量 | 在线推断 OD/葡萄糖/产物浓度 | 侧栏 → 数据分析 → 软测量模型 | 训练数据 + 实时特征 |
| 补料建议 | 基于 Monod 动力学的补料速率 | Dashboard 补料卡片 | 软测量预测值 |
| 根因分析 | 告警原因自动诊断 | 告警点击 → 根因分析 | 告警参数 + 历史数据 |

## 2. AI 对话助手

### 入口

侧栏 → **AI助手** (`/ai`)

### 功能说明

AI 对话助手支持用中文自然语言查询发酵数据。系统会将自然语言自动转换为 InfluxDB Flux 查询, 执行后返回结果。

### 使用方法

1. 打开 AI 助手页面
2. 在输入框中用中文描述查询需求
3. 系统自动调用本地 Ollama 模型 (或云端回退) 生成 Flux 查询
4. 查询结果直接展示

### 预设快捷提问

页面提供 4 个常用快捷按钮:
- **当前DO趋势分析** — 查询最近溶氧变化趋势
- **本批次与历史对比** — 对比当前批次与历史数据
- **最近异常原因** — 分析近期告警的可能原因
- **补料速率建议** — 基于当前状态给出补料建议

### 支持的查询范围

| 数据类别 | 可查字段 |
|---------|---------|
| 过程数据 (process_data) | temperature, pH, DO, pressure, airflow, rpm, weight, vfd_current 等 |
| 计算参数 (calculated_params) | OUR, kLa, mu, Vs, V_feed, V_base, V_acid, F0 等 |
| 软测量 (soft_sensor) | OD_estimated, glucose_estimated |

### 示例对话

```
用户: 查询 BATCH-20260401-001 的温度和pH趋势
AI: [自动生成Flux查询并返回数据]

用户: 最近3天DO低于20%的时间段有哪些
AI: [生成时间筛选Flux查询]
```

### 状态指示

页面右上角显示 Ollama 连接状态:
- 绿色: 本地 AI 在线
- 灰色: 本地 AI 离线 (若配置了云端 API Key, 自动回退到云端)

## 3. 对话式报告生成

### 入口

侧栏 → **AI助手** (`/ai`) → 在对话中描述报告需求

### 功能说明

通过自然语言对话, 自动生成结构化的批次分析报告。系统自动汇总批次元数据、工艺参数统计、报警记录和Phase日志, 由 LLM 生成专业的分析文本。支持迭代修改和多格式导出。

### 使用流程

#### 步骤1: 发起报告生成

在 AI 助手对话框中输入包含批次号的报告请求:

```
用户: 帮我生成 BATCH-20260401-001 的批次报告
用户: 帮我生成 BATCH-20260401-001 的批次报告, 重点分析DO偏低原因
```

系统自动识别报告意图, 开始生成。

#### 步骤2: 等待生成

报告生成需要 1-2 分钟 (依次生成 4 个章节, 每个章节调用一次 LLM)。
生成过程中对话区显示进度提示。

#### 步骤3: 预览报告

生成完成后, 右侧面板自动切换为**报告预览**, 包含:
- **章节标签导航**: 点击切换查看各章节
- **内容区域**: 渲染后的报告文本
- **导出按钮**: PDF / Word / HTML 预览

#### 步骤4: 迭代修改

报告生成后, 继续在对话框中输入修改指令:

```
用户: 加一段关于pH波动的分析
用户: 把AI建议部分写得更详细
用户: 增加一个章节, 分析搅拌策略的影响
```

系统自动识别修改意图, 更新报告内容, 右侧预览实时刷新。

### 报告章节结构

默认生成 4 个章节:

| 章节 | 内容 |
|------|------|
| **批次概况** | 批次基本信息、配方、菌株、运行时长、Phase序列、最终结果 |
| **关键参数趋势分析** | 温度/pH/DO/转速/补料的统计评价、控制精度分析 |
| **异常事件分析** | 报警记录时间线、每条报警的原因分析和影响评估 |
| **AI观察与建议** | 批次亮点、发现的问题、下一批次优化建议 |

用户可通过对话添加自定义章节 (如 "增加一个pH波动分析章节")。

### 导出格式

| 格式 | 说明 | 按钮 |
|------|------|------|
| **PDF** | A4 排版, 含封面/目录/页码, 中文字体, 打印友好 | "PDF" |
| **Word** | .docx 格式, 含标题页/目录/页码, 可二次编辑 | "Word" |
| **HTML** | 浏览器预览, 自包含样式, 可另存为网页 | "HTML预览" |

PDF 生成依赖 Puppeteer (可选安装)。若未安装, 自动回退为 HTML 文件下载。

### 关键词触发

系统通过以下关键词识别报告意图:

`报告` `报表` `report` `生成报告` `批次报告` `分析报告` `导出报告`

批次号格式: `BATCH-YYYYMMDD-NNN` (如 BATCH-20260401-001)

用户关注点通过以下模式提取:
- "重点分析..." → 该参数/现象在各章节中重点展开
- "关注..." / "特别是..." → 同上

### 数据来源

报告生成时自动拉取以下数据:

| 数据 | 来源 | 用途 |
|------|------|------|
| 批次元数据 | SQLite `batches` 表 | 封面信息、概况章节 |
| Phase 日志 | SQLite `phase_logs` 表 | 工艺流程描述 |
| 报警记录 | SQLite `alarms` 表 | 异常分析章节 |
| 状态迁移 | SQLite `state_transitions` 表 | 事件时间线 |
| 参数统计 | InfluxDB 聚合查询 | 趋势分析章节 |

### 报告存储

生成的报告保存在 `ai_sessions` 表中 (`session_type = 'report'`), 包含:
- 完整报告 JSON 结构 (`report_data` 字段)
- 对话历史 (`messages` 字段)
- 可通过 session_id 随时重新加载和继续修改

## 4. CUSUM 实时异常检测

### 入口

Dashboard 主页 → **CUSUM 实时异常检测** 面板 (自动显示)

### 功能说明

CUSUM (Cumulative Sum) 是统计过程控制领域的标准在线偏移检测算法。系统对 5 个关键工艺通道进行实时监控, 当参数发生持续性偏移 (非随机波动) 时触发告警。

### 监控通道

| 通道 | 默认基线 (均值±标准差) | 含义 |
|------|----------------------|------|
| 温度 | 37.0 ± 0.5 °C | 罐内培养温度 |
| pH | 7.0 ± 0.1 | 培养基酸碱度 |
| 溶氧 (DO) | 30.0 ± 10.0 % | 溶解氧百分比 |
| 罐压 | 0.5 ± 0.05 bar | 罐内压力 |
| 转速 | 200 ± 20 rpm | 搅拌转速 |

### 工作原理

1. **批次启动时**: 系统自动初始化 CUSUM 检测器, 设置各通道基线参数
2. **每 60 秒**: 读取 PLC 实时值, 计算归一化偏差 (σ 单位), 更新累积和
3. **异常判定**: 当正向或负向累积和超过阈值 (默认 h=5) 时触发告警
4. **批次结束时**: 自动清理检测器, 释放内存

### 面板说明

每个通道显示一行:
- **状态指示灯**: 绿色=正常, 黄色=偏移中, 红色=已告警
- **偏差值**: 显示当前归一化偏差 (±Nσ)
- **累积和条形图**: 可视化累积偏移幅度
- **重置按钮**: 告警通道可手动重置 (操作员确认偏移已处理)

### 操作指南

| 操作 | 方法 |
|------|------|
| 查看状态 | Dashboard 面板自动显示, 无需操作 |
| 重置单通道 | 点击告警通道右侧"重置"按钮 |
| 重置全部 | 点击面板右上角"全部重置" |
| 调整阈值 | API: `PUT /api/cusum/config` (见 API 参考) |

### 阈值参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| h (报警阈值) | 5 | 累积和超过 h 时触发告警, 值越大越不敏感 |
| k (漂移容许) | 0.5 | 过滤正常波动的容许带, 值越大滤掉的小漂移越多 |

**调参建议**: 初次使用建议保持默认值 (h=5, k=0.5), 运行 2-3 个批次后根据误报率调整。若误报过多, 增大 h 值; 若漏报, 减小 h 值。

## 4. 批次智能分析

### 入口

批次详情页 → **智能分析** (`/batches/{批次ID}/similar`)

### 4.1 DTW 批次相似度匹配

#### 功能说明

基于 Dynamic Time Warping (动态时间规整) 算法, 将当前批次与历史已完成批次进行时序曲线对比, 找出最相似的批次。

#### 使用方法

1. 进入任意批次详情页
2. 点击进入"智能分析"页面
3. 选择对比字段 (温度/pH/溶氧/罐压/转速)
4. 系统自动计算并展示 Top 10 最相似批次

#### 结果说明

| 列 | 含义 |
|----|------|
| 排名 | 相似度排序 (1 = 最相似) |
| 批次号 | 可点击跳转到该批次详情 |
| 配方 | 该批次使用的配方 |
| 开始时间 | 批次开始时间 |
| DTW距离 | 原始距离值 (越小越相似) |
| 相似度 | 归一化百分比 (越高越相似) |

#### 使用场景

- 新批次运行后, 找到历史上最成功的相似批次作为参考
- 异常批次诊断: 对比正常批次找出偏差点
- 配方优化: 对比不同配方版本的工艺曲线差异

### 4.2 历史包络线对比

#### 功能说明

基于同配方历史批次数据, 计算均值 ±2σ 包络线 (控制带), 将当前批次轨迹叠加显示。

#### 使用方法

1. 在智能分析页面下方, 自动显示"历史包络线对比"图表
2. 点击字段切换按钮选择查看不同参数
3. 图表交互: 支持滚轮缩放、拖拽平移

#### 图表元素

| 元素 | 说明 |
|------|------|
| 红色虚线 (上) | 历史均值 + 2σ 上限 |
| 蓝色虚线 (中) | 历史均值线 |
| 红色虚线 (下) | 历史均值 - 2σ 下限 |
| 绿色实线 | 当前批次实际轨迹 |
| 红色填充区域 | 包络线范围指示 |

#### 偏离提示

当前批次轨迹偏离包络线范围时, 图表上方会显示黄色警告: "当前批次轨迹偏离历史包络线范围"。

### 4.3 批次 AI 摘要

#### 功能说明

对已完成批次, 调用本地 LLM 自动生成 200-500 字的结构化中文摘要。

#### 摘要结构

1. **批次概况**: 菌株、配方、运行时长、结果
2. **关键工艺参数统计**: 温度/pH/DO 的均值和最大偏差
3. **重要事件时间线**: Phase 切换、报警、Hold 事件
4. **AI 观察与建议**: 异常趋势分析、与历史批次的差异、优化建议

#### 触发方式

批次详情页 → 点击"生成摘要"按钮 (仅已完成批次)

## 5. AI 自动建议系统

### 入口

Dashboard 主页 → **AI 建议卡片** (自动弹出)

### 功能说明

后台引擎每 **5 分钟** 自动分析所有运行中批次, 当检测到异常或优化机会时, 自动生成工艺调整建议。建议通过 WebSocket 实时推送到 Dashboard。

### 建议触发条件

| 类型 | 触发条件 | 置信度 | 来源模块 |
|------|---------|--------|---------|
| 溶氧过低 | DO < 20% | 80% | threshold-engine |
| 温度偏移 | \|温度 - 37°C\| > 1°C | 75% | threshold-engine |
| pH 偏移 | \|pH - 7.0\| > 0.2 | 70% | threshold-engine |
| 罐压过高 | 罐压 > 0.8 bar | 85% | threshold-engine |
| CUSUM 持续偏移 | 任一通道 CUSUM 告警 | 70% | cusum-engine |

### 建议卡片内容

每条建议包含:
- **来源模块**: threshold-engine / cusum-engine
- **目标参数**: 需调整的工艺参数
- **当前值 → 建议值**: 数值对比
- **置信度**: 0-100%
- **分析说明**: 中文描述建议原因和操作方法
- **过期时间**: 建议生成后 30 分钟自动过期

### 操作方法

| 操作 | 说明 |
|------|------|
| **接受** | 点击"接受"按钮, 建议状态变为 accepted, 记入审计日志 |
| **拒绝** | 点击"拒绝"按钮, 建议状态变为 rejected, 记入审计日志 |
| **忽略** | 不操作, 30 分钟后自动过期 |

### 去重机制

同一参数在 **30 分钟** 内不会重复生成建议, 避免建议刷屏。

### 建议生命周期

```
生成 (pending) → 接受 (accepted) → 审计记录
                → 拒绝 (rejected) → 审计记录
                → 超时 (expired)  → 自动清理
```

## 6. 软测量模型管理

### 入口

侧栏 → 数据分析 → **软测量模型** (`/analysis/soft-sensor`)

### 功能说明

软测量 (Soft Sensor) 利用易测量的在线参数 (温度、pH、DO、转速等) 推断难以在线测量的生物量参数 (OD600、葡萄糖浓度、产物浓度)。

### 6.1 训练新模型

1. 点击页面右上角 **"训练新模型"** 按钮
2. 选择目标变量:
   - **OD600 (生物量)**: 推断光密度/细胞浓度
   - **葡萄糖浓度**: 推断残留底物浓度
   - **产物浓度**: 推断目标产物浓度
3. 输入特征列表 (逗号分隔), 默认: `temperature,pH,DO,rpm`
4. 点击 **"开始训练"**
5. 系统基于历史批次数据训练线性回归模型

### 6.2 查看模型列表

模型卡片显示:
- **模型名称/ID**
- **目标变量**: OD600/glucose/product
- **R² 值**: 决定系数 (>0.8 绿色, >0.5 黄色, <0.5 红色)
- **训练样本数**
- **输入特征列表**

### 6.3 在线推断

1. 点击模型卡片上的 **"在线推断"** 按钮
2. 系统使用当前实时 PLC 数据作为输入特征
3. 返回:
   - **预测值**: 推断结果
   - **置信区间**: 95% 置信上下限 `[lower, upper]`
   - **外推警告**: 当输入特征超出训练数据范围时显示黄色警告

### 6.4 实时推送

当有活跃模型且批次正在运行时, 系统每 60 秒自动执行推断, 通过 WebSocket `soft_sensor` 通道推送到前端。

### 6.5 删除模型

点击模型卡片上的 **"删除"** 按钮, 模型立即移除。

## 7. 补料建议 (Feed Advisor)

### 入口

Dashboard 主页 → **补料建议** 卡片 (批次运行时自动显示)

### 功能说明

基于 **Monod 动力学模型** 计算最优补料速率。核心公式:

```
补料速率 F = (μ_set × X × V) / (Y_xs × S_f)
```

其中:
- μ_set: 目标比生长速率
- X: 当前生物量 (来自软测量 OD 推断)
- V: 培养液体积
- Y_xs: 底物得率系数
- S_f: 补料液浓度

### 显示内容

| 元素 | 说明 |
|------|------|
| 建议补料速率 | 计算得出的最优速率 (mL/h) |
| 动作指示 | 增加 (绿) / 减少 (红) / 维持 (灰) |
| 置信度 | 推荐可信程度 (0-100%) |
| 分析说明 | 解释推荐理由 |

### 刷新频率

每 **60 秒** 自动刷新一次。

### 默认参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| μ_max | 0.6 h⁻¹ | 最大比生长速率 (E.coli) |
| K_s | 0.05 g/L | 半饱和常数 |
| Y_xs | 0.45 g/g | 底物得率系数 |
| S_f | 500 g/L | 补料液浓度 |
| V | 5 L | 培养液体积 |

## 8. 根因分析

### 入口

告警事件 → 点击告警 → **根因分析** 弹窗

### 功能说明

当发酵过程出现告警时, 根因分析模块自动匹配已知告警模式, 分析告警前后参数趋势, 给出可能原因和建议措施。

### 使用方法

1. 在 Dashboard 告警区域点击某条告警
2. 弹出根因分析面板
3. 点击 **"开始分析"**
4. 系统返回分析结果

### 分析结果

| 部分 | 说明 |
|------|------|
| **可能原因** | 按置信度排序的原因列表, 每项附带类别标签和置信度条形图 |
| **建议措施** | 编号列表, 具体的操作步骤 |
| **分析说明** | 综合叙述性文字, 解释分析逻辑 |

### 支持的告警类型

| 告警代码 | 典型原因 |
|---------|---------|
| TEMP_HIGH / TEMP_LOW | 加热/冷却系统故障、外部温度干扰 |
| DO_LOW | 搅拌不足、通气量低、菌体高密度耗氧 |
| PH_LOW / PH_HIGH | 酸碱泵故障、代谢产物积累 |
| FOAM_HIGH | 蛋白分泌、搅拌过高 |
| CUSUM_* | 工艺参数持续偏移 |

## 11. 安全设计

### AI 建议缓冲区

所有 AI 输出均经过"建议缓冲区"机制, 确保人在回路:

```
AI分析 → ai_suggestions 表 (status=pending)
  → 前端展示给操作员
    → 操作员接受/拒绝 → 审计日志记录
```

**关键约束**:
- AI 模块 **永远不能** 直接写入 PLC 寄存器
- 所有建议需操作员明确点击"接受"后才生效
- 接受/拒绝操作均记入 `audit_logs` 表, 包含操作员 ID 和时间戳
- 未处理的建议 30 分钟后自动过期

### 建议状态追踪

```sql
-- ai_suggestions 表
id, batch_id, suggestion_type, source_module, target_param,
current_value, suggested_value, confidence, reasoning,
status (pending|accepted|rejected|expired),
created_at, expires_at, decided_by, decided_at
```

### PLC 断线保护

- PLC 通讯中断时, 前端状态机自动进入 Hold 状态
- CUSUM 检测器在无数据时不会产生误报 (检测值直接跳过)
- PLC 端独立安全连锁继续运行, 不依赖 Node.js 服务器

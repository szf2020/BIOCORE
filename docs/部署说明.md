# BIOCore 部署说明

## 1. 环境要求

- **Node.js** ≥ 20 LTS
- **pnpm** ≥ 8 (通过 corepack 启用)
- **InfluxDB** 2.7+ (本机或远程,见 `.env` 配置)
- **Ollama** (可选,用于本地 LLM)
- **PLC** (S7-200 SMART G2 或兼容设备,生产模式必须)

## 2. 安装与初始化

```bash
git clone <repo>
cd biocore
corepack pnpm install
cp .env.example .env  # 根据下方说明配置
mkdir -p packages/server/data
```

## 3. 关键环境变量

| 变量 | 开发默认 | 生产必须 | 说明 |
|---|---|---|---|
| `INFLUX_URL` | http://localhost:8086 | 真实 InfluxDB URL | 时序数据库连接 |
| `INFLUX_TOKEN` | (空) | **必填** | InfluxDB All-Access Token |
| `INFLUX_ORG` | BIOCore | 改为生产 org | |
| `INFLUX_BUCKET` | BIOCore_Data | 改为生产 bucket | |
| `JWT_SECRET` | biocore-dev-secret-... | **必填强随机** | 生成: `openssl rand -base64 32` |
| `AUTH_ENABLED` | true | true | 生产必须 true,否则无 token 自动 admin |
| `MOCK_PLC` | **true** | **false 或删除** | true 时所有 PLC 读取返回模拟值 |
| `API_V0_DEPRECATION_DATE` | 启动日+180 天 | 自定义 | 旧 /api/* 路径下线日期,过此日期可移除 v0 双挂载 |

## 4. 启动顺序

```bash
# 1. 确保 InfluxDB 已运行
curl http://localhost:8086/health  # 期望 200 OK

# 2. 启动 server (会自动跑 schema migrations)
cd packages/server
corepack pnpm exec tsx watch src/index.ts

# 3. 启动 web-ui
cd packages/web-ui
corepack pnpm exec next dev --port 3000
```

启动日志关键检查项:
- `[INFO] [Influx] 已连接 ...` — InfluxDB 连接 OK
- `[Migrator] 所有 N 个 migration 已是最新` — schema 同步 OK
- `[INFO] [API] V0 兼容期截止日期: ...` — 旧路径下线日期
- **如果看到 `⚠ MOCK_PLC=true 模式启用` 警告框,生产环境必须立即停服改 .env**

## 5. 生产部署 Checklist

### 5.1 MOCK_PLC 必须禁用
```bash
# .env 中改为
MOCK_PLC=false
# 或直接删除该行
```
开机后启动日志中**不应**出现红色警告框。

### 5.2 PLC 实机连接
- 通过 `/settings/plc-config` 添加真实 PLC 连接(S7 协议 + IP + Rack/Slot)
- 注意 `SetConnectionType(3)` 必须设置(代码已自动)
- 通过"测试连接"按钮验证可达
- 启动心跳监控

### 5.3 默认 admin 密码
首次启动会自动创建 `admin / admin123` 用户。**生产部署后必须**:
1. 用 admin/admin123 登录
2. 进入 `/settings/users` 修改 admin 密码为强密码
3. 创建其他用户分配角色 (engineer/operator/viewer)

### 5.4 API Key 创建
为 MES 等外部系统创建专用 API Key:
1. 登录后进入 `/settings/api-keys`
2. 点击"创建 API Key",填名称(例: `mes-prod`)
3. **立即复制 raw key**(只显示一次),保存到 MES 系统的环境变量
4. MES 调用 biocore 时:
   ```
   curl -H "X-API-Key: ak_xxx.xxxxx" http://biocore-host:3001/api/v1/reactors
   ```

### 5.5 数据备份
SQLite 数据库位于 `packages/server/data/biocore.db`。建议:
- 通过 `/settings/data-maintenance` 手动备份
- 或定期 `cp data/biocore.db data/backups/$(date +%Y%m%d).db`

### 5.6 反向代理 + TLS
生产环境应通过 nginx/Caddy 反向代理:
```nginx
# 例: Caddy
biocore.example.com {
  reverse_proxy /api/* localhost:3001
  reverse_proxy /ws localhost:3001
  reverse_proxy localhost:3000
}
```
WebSocket 协议使用 `wss://` (Caddy 自动处理 TLS)。

## 6. 常见问题

### 服务启动失败 "[Migrator] 致命错误"
- 检查 `packages/server/data/biocore.db` 文件权限
- 检查 `packages/server/migrations/` 目录可读
- 删除 `data/biocore.db` 后重启可全新初始化(数据全失!)

### "MOCK_PLC=true 模式启用" 警告
- 开发/演示阶段正常,**生产必须移除**

### Swagger 文档 `/api/v1/docs` 404
- 检查 swagger-jsdoc 和 swagger-ui-express 是否安装
- 检查 `apis: [__filename]` 路径正确

### WS 鉴权失败 close 1008
- 检查 `?token=` 或 `?api_key=` 是否传入
- token 是否过期(JWT 默认 24h)
- API Key 是否被撤销

### 旧 /api/* 路径仍能调用
- 这是正常的 6 个月兼容期,响应头会有 `Deprecation: version="v0", sunset="<日期>"`
- 兼容期到期后,在 `server/src/index.ts` 移除 `app.use('/api', ...)` 即可下线

## 一、项目总览

### 1.1 项目定位
BIOCore 是面向非GMP实验室R&D环境的**生物反应器智能控制平台**，基于 S7-200 SMART G2 PLC + Node.js 全栈架构。目标用户为高校发酵实验室、生物科技初创公司和合成生物学研究团队。核心卖点：以竞品 1/5 的成本提供覆盖发酵全生命周期的智能控制与数据分析平台。

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| PLC通讯 | node-snap7 (S7) + modbus-serial (Modbus RTU) |
| 后端 | Node.js 20 + TypeScript + Express |
| 状态机 | XState v5 |
| 时序数据库 | InfluxDB OSS 2.x (代码已写未部署) |
| 业务数据库 | SQLite (better-sqlite3, WAL模式) |
| 前端 | Next.js 14 App Router + React 18 + Tailwind CSS |
| UI组件 | 自建 shadcn/ui 风格 (无 Radix 依赖) |
| 拖拽 | @dnd-kit |
| 图表 | 纯 SVG (Plotly.js SSR兼容问题) |
| 本地LLM | Ollama (qwen2.5:7b) |
| 测试 | Vitest |
| 包管理 | pnpm workspace monorepo |

### 1.3 项目架构

## 九、常见问题排查

| 问题 | 解决方案 |
|------|---------|
| 前端白屏/`Cannot find module './xxx.js'` | `rm -rf packages/web-ui/.next` 然后重启 |
| PLC连接失败 errCode=665420 | 忘了 `SetConnectionType(3)`, 检查 Snap7Adapter |
| 状态机启动失败 "互锁检查未通过" | 开发模式下检查 server 中的 `devValues` 是否返回健康值 |
| 配方校验失败 "类型无效" | Phase 类型名已改为 PascalCase, 检查 recipe-validator.ts 的 VALID_PHASE_TYPES |
| `PHASE_TEMPLATES is not defined` | 配方编辑器已改为从API加载模板, 确保 `api/phase-templates` 有数据。运行 `POST /api/phase-templates/init-defaults?force=true` |
| 按钮点击延迟 | 检查是否有多余的 setTimeout/setInterval, 命令后应立即 `await fetchStatus()` |
| 端口被占用 | `netstat -ano \| grep ":3001.*LISTEN"` 找PID, `taskkill //PID xxx //F` |
| 配方下载后立即被覆盖 | `changingRecipe` 标志逻辑, 见 ControlPanel.tsx 第94行 |
| 审计对话框确认后无反应 | 检查 `/api/audit-logs` POST 是否返回 200, 失败时dialog内会显示错误 |
| Phase计时器不工作 | Phase状态需要包含 `started_at` ISO字符串, 后端 batch-controller 已设置 |

## 十一、关键约束 (违反会破坏系统)

5. **配方必须approved才能下载** — draft/superseded/archived 不可下载
6. **S7连接必须 SetConnectionType(3)**, 数据在 **DB2** (不是DB1)
7. **CIP/SIP 与生产Phase互锁** — 主发酵运行中禁止启动CIP/SIP

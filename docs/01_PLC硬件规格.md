**核心原则：PLC只做两件事——闭环控制和安全兜底。**

PLC内部不运行任何批次逻辑或配方流程。它接收来自Node.js层的设定值（SV），执行本地PID运算，并将过程值（PV）和状态字回传。即使Node.js服务崩溃或网络中断，PLC的安全连锁逻辑能够独立维持设备在安全状态。

**关键约束（来自P&ID和用户确认）：**

- **共用夹套，加热/冷却不同时工作**：蒸汽和冷却水共用同一夹套通道，通过阀门切换，任何时刻只允许一种介质通过。这意味着温控不能用分程控制（Split-Range），必须用**切换式控制（Switch Mode）**——PLC根据温度偏差方向决定当前处于"加热模式"还是"冷却模式"，并通过互锁逻辑确保蒸汽阀和冷却阀不会同时开启。
- **所有模拟量信号均为4-20mA**：温度传感器使用温度变送器输出4-20mA（非PT100直接接入），因此不需要RTD专用模块。
- **4台蠕动泵**：用于补料（葡萄糖/氮源）、补酸（HCl/H₃PO₄）、补碱（NaOH/NH₃·H₂O）、消泡剂。PLC通过DQ脉冲或模拟量控制泵速。
- **溶氧由进气量和搅拌共同作用**：DO级联控制先调搅拌转速，搅拌到上限后调空气流量。空气流量计(30NL/MIN)提供实时进氧量反馈。BIOCore提供四种可选DO策略（主动调氧/主动调补料/恒定氧气/恒定补料），温度控制与DO控制完全独立，不做温度-DO联动。

#### 罐温控制方案：工业蒸汽/冷却水切换式夹套控制

由于蒸汽和冷却水共用夹套且不同时工作，温控PID采用**切换式控制（Heating/Cooling Mode Switch）**而非分程控制。PLC内部维护一个温控模式状态字：

```
温控模式切换逻辑（PLC内部实现）：

IF (PV温度 < SV温度 - 死区)  →  进入加热模式
   关闭冷却水进阀 → 等待300ms阀门确位 → 开启蒸汽阀
   PID输出(0-100%) → 蒸汽调节阀开度(4-20mA)

ELIF (PV温度 > SV温度 + 死区)  →  进入冷却模式
   关闭蒸汽阀 → 等待300ms阀门确位 → 开启冷却水进阀
   PID输出(0-100%) → 冷却水阀开度（开关阀用PWM占空比控制）

ELSE  →  保温模式
   蒸汽阀关闭，冷却阀关闭，PID输出保持

安全互锁（硬逻辑，不可被上位机覆盖）：
   蒸汽阀开 AND 冷却阀开 = 禁止 → 两阀同时强制关闭 + 报警
   罐温 > 130°C → 蒸汽阀强制关闭 + 冷却阀强制全开
   蒸汽供气压力 < 0.5bar → 蒸汽阀禁止开启
```

**切换式控制 vs 分程控制的工程权衡：**

| 维度 | 分程控制(Split-Range) | 切换式控制(Switch Mode) |
|------|---------------------|----------------------|
| 适用条件 | 加热/冷却可同时工作（独立管路或混合阀） | **加热/冷却共用夹套，不同时工作** ✓ |
| 切换平滑性 | 高（连续过渡） | 中（需设死区避免频繁切换） |
| 安全性 | 需软件确保两阀不冲突 | **硬互锁天然保证** ✓ |
| PLC编程复杂度 | 低（单PID+信号分割） | 中（需模式判断+切换延时+双PID参数集） |

#### 搅拌控制方案：变频器RS232通讯接入PLC

搅拌电机由变频器（VFD）驱动，变频器自带RS232通讯口，通过S7-200 SMART G2的RS485/232信号板直接接入PLC。PLC通过Modbus RTU协议与变频器通讯，读取实际转速反馈并下发频率设定值，无需额外的模拟量输入/输出通道。

这种通讯控制方案相比传统的4-20mA模拟量方案有三个显著优势：精度更高（数字通讯无ADC/DAC转换误差）；可读取变频器内部诊断参数（电流、母线电压、故障码）用于预测性维护；节省了一路AO和一路AI通道。

**PLC与变频器RS232通讯参数约定：**

| 参数 | 设定值 | 说明 |
|------|-------|------|
| 通讯协议 | Modbus RTU | 变频器作为Slave，PLC作为Master |
| 波特率 | 9600 bps | 兼容大多数变频器默认设置 |
| 数据位/停止位/校验 | 8/1/Even | Modbus RTU标准配置 |
| 变频器Slave地址 | 1 | 单台变频器默认地址 |
| 频率设定寄存器 | 依品牌而定（如西门子V20: P2010/40100H） | PLC通过Modbus FC06写入目标频率 |
| 实际频率反馈寄存器 | 依品牌而定（如西门子V20: r0021/40022H） | PLC通过Modbus FC03读取实际转速 |
| 控制字寄存器 | 依品牌而定 | 启动/停止/正反转/故障复位 |
| PLC通讯周期 | 200ms | 通过STEP 7-Micro/WIN SMART的Modbus库实现 |

#### 溶氧级联控制方案：搅拌+空气流量联合作用

P&ID图纸显示罐底设有空气分布器，洁净空气经0.22μm除菌过滤器后通入罐内，由量程30NL/MIN的空气流量计实时计量。BIOCore提供四种可选的DO控制策略（详见2.6.2节），其中默认的策略一（主动调氧）采用搅拌+通气两级级联：

```
DO级联控制逻辑：

Level 1 — 搅拌调节（优先）
   DO PID输出 → 搅拌转速SV修正量
   搅拌范围: 200rpm(最低) ~ 1200rpm(上限)
   作用机理: 提高转速 → 增大气泡剪切 → 提高kLa → 提升DO

Level 2 — 通气量调节（搅拌达上限后接管）
   当搅拌已达1200rpm且DO仍低于SV：
   DO PID输出 → 空气流量调节阀开度
   空气流量范围: 1 NL/MIN ~ 30 NL/MIN
   空气流量计(4-20mA) 提供实时反馈用于流量闭环

注意: 本系统不配备纯氧混气，溶氧上限受限于空气中21%的O₂分压。
高密度发酵中如DO持续低于设定值且两级级联均达上限，
系统应触发Hold状态并提示操作员评估是否降温减缓代谢。
```

## 补充：PLC端双向心跳梯形图规格 (v2新增)

**PLC端需实现以下梯形图逻辑（STEP 7-Micro/WIN SMART编程）：**

### 心跳输出 (PLC→PC)

```
NETWORK 1 — PLC心跳输出 (T37定时器, 1秒周期)
  LD    SM0.5              // 1秒脉冲(系统位)
  EU                        // 上升沿
  MOVB  VB401_temp, VB401   // 写入当前值
  INCB  VB401_temp          // 递增(0~255自动溢出)
```

### 心跳监测 (监测PC端VB400)

```
NETWORK 2 — 监测PC心跳是否变化
  LD    SM0.5
  EU
  LDB=  VB400, VB400_last   // VB400是否与上次相同?
  INCW  VW402                // 相同→计数器+1
  LDB<> VB400, VB400_last   // VB400有变化?
  MOVW  0, VW402             // 变化→计数器清零
  MOVB  VB400, VB400_last   // 记录当前值

NETWORK 3 — 安全驻留判定 (3秒超时)
  LDW>= VW402, 3
  S     V500.0, 1           // 置位安全驻留标志
  LDW<  VW402, 3
  R     V500.0, 1           // 复位安全驻留标志
```

### 安全驻留动作

```
NETWORK 4 — 安全驻留执行
  LD    V500.0               // 安全驻留标志
  R     Q0.0, 1              // 补碱泵停
  R     Q0.1, 1              // 补料泵停
  R     Q0.2, 1              // 氮源泵停
  R     Q0.3, 1              // 补酸泵停
  MOVW  200, VW_RPM_SV       // 搅拌降至200rpm
  // 温度PID_01/02 继续运行 (不受安全驻留影响)
  // pH PID_06/07 继续运行
  // DO级联暂停, 通气维持当前AO-2输出值
```

**安全驻留恢复条件:** VB400重新变化(PC通讯恢复) → V500.0自动复位 → PLC等待上位机重新下发SV指令

## 一、PLC 硬件规格

| I/O 类型 | 数量 | 典型用途 |
|---------|------|---------|

## 二、双向心跳安全协议（Dual Heartbeat）

### 机制设计

```
PC 端（每 1 秒）  →  写入 VB400（递增计数器）  →  PLC 接收
PLC 端（每 1 秒）  →  回写 VB401（确认计数器）  →  PC 接收
```

- **监控地址：** VB400（PC→PLC 写入）/ VB401（PLC→PC 回写）
- **超时阈值：** 任一方向 **3 秒无响应** → 立即触发安全保护流程
- **断线重连：** 指数退避策略（1s → 5s → 30s），防止网络抖动引发频繁重连风暴

### 安全行为分级

| 故障场景 | 触发条件 | PLC 响应行为 |
|---------|---------|------------|
| PC 软件崩溃 | VB400 停止更新超 3s | PLC 自动 Hold 批次 + 独立维持搅拌/温控/pH |
| 网络中断 | 心跳链路断开 | 同上，PLC 进入独立安全运行模式 |
| PLC 无响应 | VB401 停止更新超 3s | PC 端报警 + 标记通讯故障 + 界面橙色警告 |
| 正常恢复 | 心跳双向恢复 | 自动从 Hold 状态恢复，继续批次执行 |

**核心价值：即使上位机完全失联，PLC 仍独立保持发酵过程安全运行，防止菌体沉降（搅拌）、防过热/过冷（温控）、防 pH 漂移。**

## 三、ISA-88 批次控制引擎

### 标准合规性

BIOCore 批次引擎遵循 **ANSI/ISA-88（IEC 61512）** 批次过程控制标准，该标准：
- 1995 年首次发布，最近一次完整修订为 2010 年
- 定义五层控制功能：配方管理 / 调度 / 顺序控制 / 调节控制 / 安全联锁
- 被 FDA 21 CFR Part 11、EU GMP Annex 11 广泛引用为推荐批次控制框架
- 西门子、ABB、Rockwell 等主要 DCS/SCADA 厂商均以 ISA-88 为批次编程基础

### 6 状态机

```
           ┌──── Hold ←──────────┐
           ↓                     │
Idle → Running → Paused ────→ Held
  ↑      ↓                       ↓
Reset  Complete ← ← ← ← ← Stopped
         ↓
       Reset → Idle
```

| 状态 | 中文 | 进入条件 | 退出条件 |
|-----|------|---------|---------|
| Idle | 闲置 | 初始 / Reset 后 | 操作员启动批次 |
| Running | 运行 | 批次正常执行中 | 完成 / 暂停 / 故障 |
| Paused | 暂停 | 操作员手动暂停 | 操作员恢复 |
| Held | 保持 | 故障监测触发 / 心跳断开 | 故障消除 + 确认恢复 |
| Stopped | 停止 | 操作员手动停止 / 紧急停止 | Reset 指令 |
| Complete | 完成 | 配方最后 Phase 执行完毕 | 操作员 Reset |

**实现技术：XState v5 状态机库**，确保状态转换的确定性和可审计性。

### 14 种 Phase 类型

| 类别 | Phase 名称 | 核心功能 |
|------|-----------|---------|
| 系统操作 | 准备（Prepare） | 检查清单验证、互锁预检 |
| 系统操作 | 加水（Water Fill） | 称重控制精确加水 |
| 系统操作 | 手动加料（Manual Add） | 操作员确认添加培养基 |
| 系统操作 | 排液（Discharge） | 批次结束排放 |
| 温控 | 升温（Heating） | 夹套加热到目标温度 |
| 温控 | 温度控制（Temp Control） | PID 温度维持 |
| 过程控制 | 搅拌（Agitation） | VFD 转速 PID |
| 过程控制 | 补料（Feeding） | 蠕动泵速率控制 |
| 过程控制 | pH 控制 | 补碱/补酸双路 PID |
| 过程控制 | DO 控制 | 搅拌-通气级联控制 |
| 过程控制 | 通气（Aeration） | 空气流量控制 |
| 发酵主体 | 发酵（Fermentation） | 全参数协同控制主阶段 |
| 清洗灭菌 | CIP 在线清洗 | 清洗液循环 + 时序控制 |
| 清洗灭菌 | SIP 在线灭菌 | F₀ 积分实时计算，验证灭菌效果 |

**配方执行模式：**
- **顺序模式（Sequential）：** Phase 自动依次推进，适合成熟标准工艺
- **自由模式（Free）：** 操作员手动控制每个 Phase，适合工艺摸索阶段
- **DAG 分支模式（v2）：** 条件路由（如"OD600 > 5 则切换补料策略"），实现工艺自适应

## 四、安全联锁体系

### 启动前互锁（IL-01 ~ IL-10）

10 项检查**全部通过**才允许启动批次，任一失败阻止启动并显示具体原因：

| 互锁编号 | 检查项目 | 检查逻辑 |
|---------|---------|---------|
| IL-01 | 传感器信号有效性 | pH/DO/温度传感器读数在量程范围内 |
| IL-02 | VFD 通讯就绪 | Modbus RTU 通讯响应正常 |
| IL-03 | 急停复位确认 | 急停按钮处于释放状态 |
| IL-04 | 盖锁到位 | DI 信号确认发酵罐盖已锁定 |
| IL-05 | 蒸汽压力就绪 | 蒸汽供应压力在设定范围 |
| IL-06 | PLC 心跳正常 | 双向心跳通讯已建立 |
| IL-07 | 校准有效性 | pH/DO 校准日期未过期 |
| IL-08 | 冷却水供应 | 冷却水流量/压力确认 |
| IL-09 | 配方版本审批 | 配方处于"已批准"状态 |
| IL-10 | 报警清零确认 | 无未确认的活跃报警 |

### 运行故障监测（RF-01 ~ RF-11）

批次运行中持续监测，触发条件满足 → **自动切换到 Held 状态**保护批次：

| 故障编号 | 监测参数 | 触发条件 | 响应动作 |
|---------|---------|---------|---------|
| RF-01 | 温度偏差 | 偏差 >2°C 持续 3 分钟 | 自动 Hold + 报警 |
| RF-02 | pH 失控 | pH 超设定范围 5 分钟 | 自动 Hold + 报警 |
| RF-03 | DO 过低 | DO <5% 持续 5 分钟 | 自动 Hold + 报警 |
| RF-04 | 罐压超限 | 罐压超设定上限 | 自动 Hold + 报警 |
| RF-05 | 搅拌故障 | VFD 报错或转速偏差过大 | 自动 Hold + 报警 |
| RF-06 | 通讯断线 | PLC 心跳丢失 >3s | 自动 Hold + PLC 独立运行 |
| RF-07 | 传感器超量程 | 任一 AI 读数超量程 | 自动 Hold + 报警 |
| RF-08 | 补料泵故障 | DI 确认信号异常 | 自动 Hold + 报警 |
| RF-09 | 泡沫检测 | 称重突变 >5%（非泵料期间） | 自动启动消泡泵 |
| RF-10 | 冷却失效 | 温度持续上升超设定斜率 | 自动 Hold + 紧急冷却 |
| RF-11 | 紧急停止 | DI 急停按钮触发 | 立即停止所有执行机构 |

## 五、安全设计亮点总览

### 1. AI 建议缓冲区（Human-in-the-Loop）

```
AI 推理引擎
    ↓
生成建议（参数值 + 置信度 + 推理依据）
    ↓
写入建议缓冲区（pending 状态，不执行）
    ↓
界面展示 → 操作员审核（Accept / Reject）
    ↓（仅 Accept 后）
写入 PLC 设定值寄存器
```

**设计原则：AI 永不直接控制 PLC。** 所有 AI 建议必须经人工确认方可执行。这使 BIOCore 区别于全自动闭环 AI 系统，在实验室安全规范框架内实现 AI 辅助。

### 2. 不可篡改审计日志

- 基于 SQLite 数据库触发器实现：禁止对日志表执行 UPDATE / DELETE 操作
- 所有操作（配方变更、参数调整、报警确认、AI 建议接受/拒绝）永久记录
- 支持 CSV 导出，满足 GMP/研究机构数据完整性要求
- 符合 FDA 21 CFR Part 11 电子记录不可篡改原则（非 GMP 场景的前瞻性设计）

### 3. F₀ 灭菌积分验证

SIP 阶段实时计算灭菌当量 F₀（分钟），确保灭菌效果达标：
- F₀ 目标值：≥ 20 分钟（对应 12 个 D 值，对 BI 孢子达到 10⁻¹² 存活概率）
- 实时图表展示积分曲线，灭菌完成后自动记录达标时刻

### 4. 安全表达式求值器

配方 DAG 分支条件（如 `OD600 > 5`）使用：
- 白名单 AST（抽象语法树）解析器
- 仅允许预定义操作符和过程变量
- 杜绝任意代码注入攻击，防止配方文件被恶意利用

### 5. API 安全层

- JWT 令牌 24 小时过期，防止长期凭证泄露
- API Key 使用哈希存储，原始值不落库
- 时序安全比较（constant-time comparison）防止时序攻击
- 文件上传验证 MIME + 魔术字节 + 20MB 大小限制

## 七、行业标准对比背景（外部参考）

### ISA-88 / IEC 61512 批次控制标准

- BIOCore 批次引擎与 ISA-88 状态机定义完全对齐，具备工业合规基础
- ISA-88 Equipment Module 概念对应 BIOCore 的 Phase 层，提供设备独立的互锁和允许逻辑
- Siemens PCS 7、Rockwell FactoryTalk Batch 等工业级 SCADA 均基于相同标准实现
- 参考：[ISA-88 标准主页](https://www.isa.org/standards-and-publications/isa-standards/isa-88-standards)

### IEC 61511 功能安全标准（过程工业）

BIOCore 的安全设计借鉴了 IEC 61511 的核心概念，但作为 R&D 实验室产品不要求全面 SIL 认证：

| IEC 61511 概念 | BIOCore 对应实现 |
|--------------|----------------|
| 安全仪表系统（SIS）独立性 | PLC 独立安全连锁，与 PC 上位机解耦 |
| 通讯看门狗（Watchdog） | 双向心跳 VB400/VB401，3 秒超时 |
| 安全完整性等级（SIL） | 互锁逻辑覆盖 10 项启动检查 + 11 项运行监测 |
| 操作员确认要求 | AI 建议缓冲区，所有 AI 动作需人工批准 |
| 审计追踪 | 不可篡改审计日志（SQLite 触发器） |

IEC 61511:2016（最新版本 2026 年即将更新）将 SIL 1 定义为 10 倍风险降低，SIL 2 为 100 倍。BIOCore 的多层防护设计（互锁 + 心跳 + 独立 PLC 安全程序）在 R&D 场景下提供与 SIL 1 相当的保护深度。

参考：[IEC 61511 标准页](https://webstore.iec.ch/en/publication/24241) | [功能安全标准对比 exida](https://www.exida.com/blog/functional-safety-standards-iec-61508-vs.-iec-61511)
